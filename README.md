# HabitatOne - Society Management System MVP

HabitatOne is a clean, simple, and maintainable residential society administration platform that digitizes day-to-day operations like resident registration, notice boards, maintenance billing, complaints tracking, and visitor logging.

## Tech Stack
* **Backend**: Python 3.11+, Django 5.2.1, Django ORM
* **Database**: SQLite (default, easily portable to PostgreSQL)
* **Frontend**: HTML5, Django Template Engine, Vanilla CSS, Bootstrap 5, Bootstrap Icons
* **Authentication**: Built-in Django Authentication System with role-based view enforcement

---

## Directory Structure
```
society_management/
│
├── accounts/          # Login, logout, roles, and custom User model
├── residents/         # Resident profile creation & CRUD (Admin only)
├── notices/           # Notice board announcements
├── maintenance/       # Maintenance billing & dues settlement
├── complaints/        # Resident complaint logging & resolution tracking
├── visitors/          # Pre-registering visitor entries
├── dashboard/         # Role-based KPI stats & metrics
│
├── templates/         # Master layouts & app HTML templates
├── static/            # custom.css and general static assets
│
├── society_management/ # Core Django project configurations
├── manage.py
├── seed_data.py       # DB Seeding script for sample records
└── db.sqlite3
```

## Getting Started

### 1. Run migrations to initialize the database
Since the migrations are already generated, you can apply them by running:
```bash
python manage.py migrate
```

### 2. Seed the database with sample data
To load the system with dummy records (admin, residents, sample bills, notices, and complaints), execute the seeding script:
```bash
python seed_data.py
```

### 3. Start the development server
Run the Django local development server:
```bash
python manage.py runserver
```
Once running, visit the portal in your browser at `http://127.0.0.1:8000/`.
